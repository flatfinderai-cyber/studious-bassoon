## PHASE 4 — CORE FEATURE BUILDS

### Task 4.1 — VMC Quality Checker

**Location:** `/packages/core/lib/vmc-quality.ts`

This module runs server-side only. It is never imported client-side.

```typescript
export interface VMCCheckResult {
  passed: boolean;
  wordCount: number;
  dictionaryScore: number;
  semanticScore: number;
  uniquenessScore: number;
  rejectionReason?: string;
}

export async function checkMessageQuality(
  message: string,
  threadHistory: string[],
  propertyContext: { address: string; rent: number; availableDate: string }
): Promise<VMCCheckResult>
```

Rules:
- Word count minimum: 40. Reject immediately if under.
- Dictionary integrity: Anthropic API score — "Score this message 0-100 for the percentage of words that are real English dictionary words. Return only a number." Reject if under 85.
- Semantic coherence: Anthropic API — "This message was sent in a rental conversation about a property at [address] for [rent]/month available [date]. Score 0-100 for topical relevance to a genuine rental enquiry. Return only a number." Reject if under 60.
- Uniqueness: string similarity against all previous messages in thread. Reject if similarity to any prior message exceeds 60%.
- Template detection: maintain array of known filler phrases. Reject if message is composed predominantly of these.

### Task 4.2 — VMC Response Timer

**Location:** `/packages/core/lib/vmc-timer.ts`

Supabase Edge Function running every 5 minutes:
- Threads where landlord has not responded and deadline is within 12 hours → send reminder
- Threads where deadline is within 4 hours → send final warning, flag listing internally
- Threads where deadline has passed → suspend listing, notify tenant with priority re-matching, update response rate, log strike, check escalation

### Task 4.3 — Discrimination Scanner

**Location:** `/packages/core/lib/discrimination-scanner.ts`

Server-side only.

```typescript
export interface ScanResult {
  passed: boolean;
  flagType: 'clean' | 'discrimination' | 'predatory';
  confidence: number;
  triggeringPhrase?: string;
  rcmpReferralRequired: boolean;
}

export async function scanListingContent(
  listingText: string,
  landlordId: string
): Promise<ScanResult>
```

Anthropic API system prompt:
```
You are a housing discrimination and predatory listing detection system operating under the Ontario Human Rights Code and Canadian Human Rights Act. Analyse the provided listing text and return a JSON object with:
- flagType: "clean" | "discrimination" | "predatory"
- confidence: 0-100
- triggeringPhrase: the exact phrase that triggered the flag, or null
- rcmpReferralRequired: true if the content suggests human trafficking recruitment patterns, non-monetary compensation for tenancy, or grooming/coercion language

Discrimination includes: gender specification, age specification, ethnic or national origin indicators, family status discrimination, any language implying a preferred tenant type based on a protected ground.

Predatory includes: nominal or dollar-figure rent tied to personal characteristics, ambiguous compensation structures, urgency combined with vulnerability targeting.

Return only valid JSON. No other text.
```

Consequence logic:
- `clean`: listing proceeds
- `discrimination`: block listing, permanent ban, Do Not Fly List entry, notify landlord of suspension only
- `predatory`: block listing, permanent ban, Do Not Fly List entry, RCMP referral package auto-compiled
